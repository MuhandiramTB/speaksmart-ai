# Story 1.2–1.5 — Voice Loop (Capture → STT → Chat → TTS)

**Status:** In Progress
**Epic:** 1 — Foundation & Voice Loop
**Kiro Reqs:** R1.1–1.6, R2.1–2.5, R3.1–3.4, R4.1–4.4

## As a / I want / So that
**As a** student, **I want** to press a button, speak, and hear the AI reply, **so that** I can practice spoken English in a real conversation.

## Acceptance Criteria
1. Mic button records audio in browser.
2. Audio is transcribed via Groq Whisper and shown in chat.
3. Transcript is sent to Groq Llama 3.3 and streamed back.
4. AI reply is spoken via browser TTS.
5. Mute toggle for TTS.

## Dev Notes
- Use `MediaRecorder` with `audio/webm;codecs=opus`.
- Groq `audio.transcriptions` requires a `File` — wrap the Blob.
- For chat streaming, return a `ReadableStream` from the Route Handler.
- TTS via `window.speechSynthesis` — no API needed.
